How Brief works

No install. The skill travels inside every brief, so any agent with local file access can act on it.

1

Name a brief, then click Record on it — talk through the bug or task while showing it on screen, then hit stop. It saves locally to ~/Downloads/brief/ as a self-contained zip (transcript, screenshots, and the filing skill inside).

2

Stack up as many as you like in your list.

3

Hit Export and paste the prompt into your coding agent. It reads the skill bundled in the brief, files a real ticket in your tracker, and deletes that brief once it's filed.

✦

Works with any local agent. Claude Code Desktop, Cursor, Cline — anything that can read files in ~/Downloads/brief/ and reach your tracker. The prompt points it at skill/SKILL.md inside the brief.

Everything is local. Briefs never leave your machine until an agent — running on your machine — reads them. You'll need a tracker connected to that agent (Linear, GitHub, Jira, Notion).
